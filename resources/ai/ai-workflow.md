# AI Workflow for Software Engineering

## Philosophy

AI should not be used as a simple code generator where we copy-paste the output and hope that it works.

As engineers, we use AI as a tool to improve our workflow:

- Solve problems faster
- Explore better design decisions
- Review and improve our code
- Debug issues more efficiently
- Learn complex concepts deeper

The goal is not to replace engineering thinking, but to enhance it.

---

# Useful AI Prompts

## Code Review

```
Act as a senior backend engineer.

Review this code.

Analyze:

- Bugs
- Readability
- Maintainability
- Performance
- Security issues

Explain your reasoning and suggest improvements.
```

---

## Learning

```
Explain this concept as a backend engineer.

Include:

- Definition
- Why it matters
- Real-world examples
- Common mistakes
- Interview perspective
```

---

## Architecture Design

```
Act as a software architect.

Help me design this system.

Discuss:

- Requirements
- Components
- Database design
- Trade-offs
- Scalability concerns

Explain the reasoning behind each design decision.
```

---

## Engineering Principle

AI is a collaborator, not a replacement for understanding.

Every solution should be reviewed, understood, and validated before being implemented.
